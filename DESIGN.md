# KeyboardPet - 键盘宠物 MVP 设计文档

## 项目概述

KeyboardPet 是一个 macOS 原生桌面宠物应用，通过监听用户的键盘输入行为来驱动宠物的状态变化、成长和互动。它不是一个被动的装饰品，而是一个能感知你编码节奏的伙伴。

### 核心定位

**用键盘行为驱动养成 + 数据洞察**

与现有项目的差异：
- vs Bongo Cat：不只是模仿打字动作，而是有完整的状态/成长/养成系统
- vs Mac Pet：驱动源是真实键盘输入，不是番茄钟计时器
- vs Clawd on Desk：关注的是"程序员自己在干嘛"，而非 AI agent 状态

### 设计参考

| 参考项目 | 借鉴点 |
|----------|--------|
| Bongo Cat Mac Keyboard | 键盘监听实现方式（CGEventTap） |
| Mac Pet | 菜单栏集成、活动可视化（贡献图风格）、低资源占用 |
| Clawd on Desk | 像素动画状态机、多状态切换、眼睛追踪、睡眠序列 |

## MVP 功能范围

### P0 - 必须有（MVP 核心）

#### 1. 键盘监听引擎
- 使用 CGEventTap 监听全局按键事件
- 仅记录 keyCode（物理键位）+ 时间戳，**绝不记录实际输入内容**
- 计算实时指标：
  - WPM（每分钟按键数）
  - 删除率（Delete/Backspace 占比）
  - 空闲时长（距上次按键的秒数）
  - 连续编码时长
- 首次运行引导用户授予辅助功能权限

#### 2. 宠物状态机
- 宠物在屏幕上以透明悬浮窗口显示
- 基于键盘指标自动切换状态（详见状态机设计）
- 每个状态对应不同的视觉表现

#### 3. 基础渲染
- 透明悬浮窗口（NSWindow, level = .floating, 穿透点击, 可拖拽）
- 默认位置：屏幕右下角，记住上次位置
- 跟随主显示器
- 卡通像素风格角色，使用 SwiftUI Canvas 绘制
- 基础动画过渡（状态切换时平滑变化）

#### 4. 菜单栏入口
- MenuBarExtra 提供快捷操作
- 显示当前状态摘要（今日击键数、当前 WPM）
- 退出应用

### P1 - 应该有（MVP 增强）

#### 5. 经验值与等级
- 每日击键量累积经验值
- 升级解锁新表情/动画
- 等级持久化存储

#### 6. 每日统计面板
- 从菜单栏打开统计窗口
- 今日击键数、活跃时长、峰值 WPM
- 按小时分布的活动热力图
- 最活跃时段标记

#### 7. 行为彩蛋
- 连续快速删除 → 宠物焦虑
- 深夜编码（0:00-5:00）→ 宠物穿睡衣
- WPM 破纪录 → 宠物庆祝
- 长时间无输入后恢复 → 宠物惊醒

### P2 - 可以有（后续迭代）

- 像素美术资源替换几何图形
- 宠物日记（AI 生成第一人称日报）
- 成就徽章系统
- 多宠物 / 皮肤系统
- 周报导出 / 分享卡片
- git commit/push 事件集成
- 眼睛追踪鼠标（参考 Clawd on Desk）

## 宠物状态机设计

### 状态定义

```
┌─────────────────────────────────────────────────────┐
│                    宠物状态机                         │
├──────────┬──────────────────┬────────────────────────┤
│ 状态      │ 触发条件          │ 视觉表现              │
├──────────┼──────────────────┼────────────────────────┤
│ idle     │ 无输入 < 30s      │ 站立，偶尔眨眼         │
│ typing   │ WPM > 0          │ 手臂摆动，跟随节奏      │
│ flow     │ WPM > 80 持续 30s │ 发光特效，兴奋表情      │
│ deleting │ 删除率 > 50%      │ 冒汗，焦虑表情         │
│ thinking │ 无输入 30s-120s   │ 托腮思考，头顶问号      │
│ sleepy   │ 无输入 120s-300s  │ 打哈欠，眼皮下垂       │
│ sleeping │ 无输入 > 300s     │ 躺下，冒 zzZ 气泡      │
│ wakeup   │ 从 sleeping 恢复  │ 惊醒弹跳，2s 过渡动画  │
│ night    │ 系统时间 0:00-5:00│ 穿睡衣（叠加状态）      │
│ record   │ WPM 破个人纪录    │ 放烟花，3s 庆祝动画     │
└──────────┴──────────────────┴────────────────────────┘
```

### 状态转换图

```
                    ┌──────────┐
           按键恢复  │ sleeping │ 无输入 > 300s
           ┌───────▶│  💤 zzZ  │◀───────────┐
           │        └──────────┘             │
           │             │                   │
           │         按键恢复                 │
           │             ▼                   │
           │        ┌──────────┐             │
           │        │  wakeup  │             │
           │        │  😲 !    │             │
           │        └────┬─────┘             │
           │             │ 2s后              │
           │             ▼                   │
      ┌────┴─────┐  ┌──────────┐  ┌─────────┴──┐
      │ thinking │  │   idle   │  │   sleepy   │
      │  🤔 ?   │◀─│   😊    │─▶│   😪 ~    │
      └────┬─────┘  └────┬─────┘  └────────────┘
           │         30s无输入     120s无输入
      有按键│              │
           │         有按键│
           ▼              ▼
      ┌──────────┐  ┌──────────┐
      │  typing  │◀▶│  typing  │
      │  ✋ ⌨️   │  │  ✋ ⌨️   │
      └────┬─────┘  └──────────┘
           │              │
     WPM>80│        删除率>50%
     持续30s│              │
           ▼              ▼
      ┌──────────┐  ┌──────────┐
      │   flow   │  │ deleting │
      │  🔥 ✨   │  │  😰 ⌫   │
      └──────────┘  └──────────┘

      ※ night 状态为叠加层，不影响主状态流转
      ※ record 状态为临时触发，3s 后回到原状态
```

### 状态优先级

当多个条件同时满足时，按以下优先级判定：

1. `record` — 最高优先级，临时覆盖
2. `wakeup` — 过渡状态，固定时长
3. `flow` — 进入心流状态
4. `deleting` — 大量删除
5. `typing` — 正常打字
6. `thinking` — 短暂停顿
7. `sleepy` — 较长停顿
8. `sleeping` — 长时间无活动
9. `idle` — 默认状态

## 技术架构

```
KeyboardPet.app (macOS native, Swift)
│
├── App Layer
│   ├── KeyboardPetApp.swift          // SwiftUI App 入口
│   ├── MenuBarManager.swift          // 菜单栏 MenuBarExtra
│   └── PetWindow.swift               // 透明悬浮窗口管理
│
├── Core Layer
│   ├── KeyboardMonitor.swift          // CGEventTap 键盘监听
│   ├── MetricsEngine.swift            // 实时指标计算 (WPM, 删除率等)
│   ├── PetStateMachine.swift          // 状态机引擎
│   └── ExperienceManager.swift        // 经验值/等级管理 (P1)
│
├── View Layer
│   ├── PetView.swift                  // 宠物渲染 (SpriteKit or SwiftUI Canvas)
│   ├── PetAnimations.swift            // 动画定义
│   └── StatsPanel.swift               // 统计面板 (P1)
│
├── Data Layer
│   ├── StatsStore.swift               // 击键统计持久化 (SwiftData)
│   └── UserDefaults+Extensions.swift  // 轻量配置存储
│
└── Resources
    └── Assets.xcassets                // 图标、颜色资源
```

### 关键技术选型

| 模块 | 方案 | 理由 |
|------|------|------|
| 键盘监听 | CGEventTap | macOS 最成熟的全局事件监听方案，参考 Bongo Cat 实现 |
| UI 框架 | SwiftUI + AppKit 混合 | SwiftUI 做菜单栏和面板，AppKit 管理悬浮窗口 |
| 宠物渲染 | SwiftUI Canvas | 卡通像素风格，Canvas 绘制足够且轻量；后续可换 SpriteKit |
| 数据存储 | SwiftData | 击键统计持久化，macOS 13+ 原生支持 |
| 菜单栏 | MenuBarExtra | macOS 13+ 原生 API，无需第三方库 |

### 隐私设计

- **仅记录 keyCode + timestamp**，不记录键值内容
- 不记录任何窗口标题、应用名称、输入法状态
- 所有数据仅存储在本地，不联网
- 辅助功能权限声明清晰告知用途

### 性能目标

- CPU 占用 < 1%（参考 Mac Pet 标准）
- 内存占用 < 30MB
- 键盘事件处理延迟 < 1ms，不影响正常打字体验
- 统计数据每 60s 批量写入，避免频繁 I/O

## MVP 里程碑

| 阶段 | 内容 | 预估 |
|------|------|------|
| M1 | 项目骨架 + 键盘监听 + 终端输出验证 | 基础搭建 |
| M2 | 透明窗口 + 几何宠物 + idle/typing 两个状态 | 核心体验 |
| M3 | 完整状态机（8 个状态 + 过渡动画） | 丰富互动 |
| M4 | 菜单栏入口 + 基础统计显示 | 实用性 |
| M5 | 经验值系统 + 每日统计面板 | 养成感 |

## 设计决策（已确认）

1. **宠物形象**：自行设计卡通像素风格角色，直接在 MVP 中使用
2. **窗口位置**：默认右下角，记住上次位置（持久化存储）
3. **多显示器**：跟随主显示器
4. **系统版本**：最低支持 macOS 14 (Sonoma)
5. **分发方式**：自用开发，后期开源发布到 GitHub（不上架 App Store，无沙盒限制）
6. **窗口形式**：悬浮透明窗口，类似 Clawd on Desk 效果（置顶、透明背景、穿透点击、可拖拽）
